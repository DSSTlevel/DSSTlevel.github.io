Route Cause Analysis

Route Cause Analysis - The process of identifying the original fauilt through analysising factors that had followed on from the original issue
Fish Bone Diagram - Visual understanding of the root cause analysis route. Assorting ideas into categories
How to create a Fishbone Diagram? Agree on a problem statement
Idenfity the main categories to be inserted into the diagram
Write the main categories to be decomposed down into small problems
Identify all the possibilities that had originally caused the issue.
Final analysis of the problem to see if the search matches
Example
An example of this would be to understand the original cause of a data leakage found from a database through identifying an alternative solution and decomposing it down to see if the route matches the issue.
Concept Testing - Understand the advantages and drawbacks of potential improvement for a particular concept
Five Whys - Repetition of 5 "why" questions to delve further into the issue, to try and find the root cause

Testing Types

Functionality Testing - Confirming of component functionality and readiness against requirements
Usability Testing - Verification of component functionality and readiness to release
Compatability Testing - Confirm that components can work alongside with software and hardware
Accessibility Testing - Form of usability testing that checks the inclusion of the services/products to the end user
Customer/End User Satisfaction Survey - Confirm the experience mood of the end user, whether this be positive or negative
Fault Finding and Debugging - Finding to search for the bugs and errors, then debug/fix these bugs and errors
Impact Assessment - Implications for a business, the environments and people of the implementation of components
Efficiency of individual components - Tests the performance of a component and its resources required for its function
Review accuracy of data - Review of data to test if there is bad data in the business
Ensuring desired outcomes - Making sure the service/product meets the needs of the end user, business, and employees
Performance monitoring - Acts as a warning system that reports to the IT team when something has "gone wrong"
